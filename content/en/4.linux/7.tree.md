## Installing `tree`

| System / OS                                     | Installation / Availability                                                           |
| ----------------------------------------------- | ------------------------------------------------------------------------------------- |
| **macOS**                                       | via Homebrew: `brew install tree`                                                     |
| **Linux / Unix** (Debian, Ubuntu, Fedora, etc.) | via the package manager: e.g., `sudo apt-get install tree` or `sudo yum install tree` |
| **Windows** (Command Prompt / cmd.exe)          | the `tree` command is natively available in the Command Prompt                        |
| **Other Unix / BSD**                            | often available via the package manager or by compiling the tool                      |

## `tree` Command

The `tree` command displays the directory structure in a readable format in the console (subdirectories, files, depth, etc.).
It is available on multiple operating systems:

### Usage Example

```sh
# simple tree structure (max depth 3)
tree -L 3

# including hidden files, excluding patterns (e.g., venv, .git)
tree -L 3 -a -I 'venv|.git|__pycache__'

# with readable sizes and dates
tree -L 3 -a -h -D -I 'venv|.git|__pycache__'

# only directories (depth 2)
tree -d -L 2

# display full paths
tree -f -L 3

# redirect output to a file
tree -L 4 -a -I 'venv|.git|__pycache__' > tree.txt
```
