## Basic commands

### Connection

```shell
ssh <USERNAME>@<IP_VPS> # Standard
ssh -i ~/.ssh/<KEY_NAME>.pub <USERNAME>@<IP_VPS> # With SSH key
```

#### Using an alias

Create an alias in the SSH config file `~/.ssh/config`. Provide the private key path.

```shell [.ssh/config]
Host <ALIAS_NAME>
    Hostname <IP_VPS>
    User <USERNAME>
    IdentityFile ~/.ssh/<KEY_NAME>
    IdentitiesOnly yes
```

```shell [Connect using an alias]
ssh <ALIAS_NAME>
```

### Setup

Details for the `ssh-copy-id` command to configure SSH access on the server:

1. Copies the public key to the remote server.
2. Sets appropriate permissions automatically.
3. Appends the key to the `authorized_keys` file on the remote server.

```shell
ssh-copy-id -i ~/.ssh/<key_name>.pub <username>@<ip_vps>
```

### Copy files

```shell [Copy a file over SSH]
scp -r <FILE_NAME> <username>@<ip_vps>:~/
```

```shell [Copy a file over SSH on a custom port]
scp -r -P <PORT> <FILE_NAME> <username>@<ip_vps>:~/
```

### Remove

```shell [Remove a host key from known_hosts]
ssh-keygen -R <IP_VPS>
```

## Connect to a server

::Steps{level="3"}

### Generate an SSH key

Generate an SSH key pair on your local machine.

```shell
ssh-keygen -t rsa -b 4096 -C "email@example.com" -f ~/.ssh/<key_name>
```

### Add the public SSH key to Pulseheberg

You can copy it with the following command (macOS):

```shell
pbcopy < ~/.ssh/<key_name>.pub
```

### Connect to the VPS

From your local terminal, connect to the VPS with:

```shell
ssh -i ~/.ssh/<key_name> root@<ip_vps>
```

::

## Key management

Authorize the key by appending it to the `authorized_keys` file:

```shell
echo "<public_key>" >> ~/.ssh/authorized_keys
```
